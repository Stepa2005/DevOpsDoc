**Go Resolver**
Go использует или собственный “Go” resolver, или же “Cgo” (больше деталей можно найти в [оф. документации](https://pkg.go.dev/net#hdr-Name_Resolution)). Главное отличие данных реализаций состоит в том, что **Go resolver** построен на горутинах и использует свою логику для резолвинга, построенную на базе **RFC**, в то время как в **Cgo** осуществляется синхронный системный вызов API glibc в отдельном потоке. 

*Под капотом Go написано много кода для определения какой resolver использовать в том или ином окружении.* В основном там идет проверка на возможность интерпритации опции в `/etc/resolv.conf` и `/etc/nsswitch.conf`. Если есть хоть какие-то неподдерживаемые Go resolver’ом опции, то будет использоваться Cgo resolver, если он принудительно не выключен, либо совсем не поддерживается.

Если выбран **“Go” resolver**, то в самом стандартном варианте для разрешения static хостов используется `/etc/hosts` файл, затем `/etc/resolv.conf`для выполнения запросов к dns серверу. И всё. Хочу обратить внимание, что Go **не имеет** никакой внутренней логики по кэшированию dns запросов

Итак мы находимся в точке, когда запрос на разрешение имени хоста попал к сервису **systemd-resolved**.
